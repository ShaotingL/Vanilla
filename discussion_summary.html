核心討論脈絡

主題修改內容原因
研究目標定義把目標收斂為利用公共 RNA-seq reads 重新 mapping 到 CR0040_HaplotypeA，找 rostellum 發育與 MADS 調控候選基因。重點不是複製原文，而是找可驗證、可操作的 rostellum candidate genes。
資料來源釐清本階段只使用公共定序檔案與自己的 CR0040A mapping/count matrix；原文 HPP92/KAG、WGCNA module 先保留作 hold-out comparison。避免把原文結果混進 ranking，讓 CR0040A 分析能獨立評估。
Sample design 解釋把 BGR、B、GR、G 解剖代號寫清楚，並加入原文 Figure 5 sampling 圖與圖說。原本 tissue code 不易理解，因此改從取樣邏輯解釋每個 contrast。
QC 與樣本排除列出所有 SRR/sample/stage/tissue/replicate，並排除 SRR24981847 / B_S2_A。該樣本 assigned reads 只有 462，STAR unique mapping 約 0.06%。
差異表現分析加入 PyDESeq2 / DESeq2-style contrast summary 與 padj 欄位。使用者要求不要停在 fold-change prioritization，而要有正式統計支撐。
MADS bait 定義MADS-like genes 從文字註解搜尋改為 OrchidBase7 Pfam evidence：PF00319/PF09047 定義 MADS core，PF01486 只作 K-box support。使用者指出 MADS 應從 Pfam 註解篩選。
BLASTP 取捨不把原文 Arabidopsis MADS BLASTP homolog search 納入目前 ranking。目前目標是建立 CR0040A 上可直接重現的 domain-supported MADS bait；BLASTP 留作後續分類或 ortholog cross-check。
OrchidBase7 predicted targets針對重要 MADS bait 抓 OrchidBase7 TF target prediction，並和第一波 qPCR 驗證候選 / top 500 ranked genes 取交集。使用者希望不只看 co-expression，也利用網站工具找可能 target genes。
原文 Table 1 比較讀入原文 Table 1，新增 CR0040A MADS 數量與 protein length distribution 比較。比較我們 Pfam 篩出的數量與原文 47 個 VpMADS genes。
Ash + Vanilla MADS 演化樹使用 AshMADS-box.pep 中實際存在的 36 條序列與 CR0040A 42 條 MADS core 建立 MAFFT + IQ-TREE2 tree。使用者決定不補版本不一致的 Apostasia 缺失序列，直接用現有序列。
實驗邏輯流程圖新增 Experiment Logic Flowchart，串起研究問題、資料來源、contrast 設計、統計、MADS/target 篩選與驗證。讓讀者先理解為什麼這樣分析。
S3/S4 判斷來源改成明確說明：論文提到 S3/S4 同時有 G 與 GR，因此本分析選 S3/S4 做主要 GR_vs_G；不是論文直接宣稱 S3/S4 才是 rostellum 形成期。區分原文取樣事實與本分析推論。
DESeq2 contrast 說明補充 BGR_vs_B 有 S1-S4，GR_vs_G 只有 S3/S4 的原因。兩者取決於哪些 stage 同時有成組樣本。
qPCR shortlist 定義改寫為「第一波 qPCR 驗證候選名單」，說明它是從 CR0040A RNA-seq ranking 篩出的 50 個候選，不是外部資料或原文清單。避免讀者不知道 qPCR list 從哪裡來。
Heatmap 候選基因來源Figure B 下方新增 heatmap genes 表，列出前 35 個候選的 ID、score、S3/S4 GR_vs_G、BGR_vs_B、CArG 與 annotation。使用者詢問 heatmap 裡的候選基因到底是什麼。
OrchidBase7 target set 解釋把 broad target set 門檻改成白話說明：target 數量超過 20,000 時名單太寬，不適合用來縮小候選；只把 target 數量 ≤ 10,000 的 MADS 放進交集分析。使用者指出原本「target set 不過大」難理解。

重要結論

目前 CR0040A data 中最強的 local MADS regulator candidate 是 VANPL_A_00001t031260（SVP-like）：local MADS bait score 最高，S3 GR_vs_G log2FC 為正且有統計支持，並與 OrchidBase7 predicted target overlap 形成可驗證的下游候選。

這仍不是最終功能證明。下一步應先做 qPCR 驗證 S3/S4 的 G vs GR 與 B vs BGR，再選擇 MADS 或下游 target 做 VIGS/RNAi/functional assay。
