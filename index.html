CR0040A Vanilla Rostellum MADS Target Analysis

目的：利用公共 Vanilla rostellum RNA-seq reads，重新 mapping 到 CR0040_HaplotypeA，找出可能影響 rostellum 發育、適合後續 qPCR / VIGS / functional assay 的 MADS 與下游候選基因。

目前結論

38QC 後納入分析的 RNA-seq samples
26,392CR0040A genes ranked
42Pfam-supported MADS core genes
50第一波 qPCR 驗證候選 genes
目前最值得優先追的 MADS candidate：VANPL_A_00001t031260（SVP-like）。它在 CR0040A data 中的 local MADS bait score 最高，S3 GR_vs_G 表現上升且有統計支持，也有 OrchidBase7 predicted target overlap 可以往下游候選延伸。

這不是最終功能證明，而是候選排序。真正要回答「抑制 rostellum 生長是否能降低人工授粉成本」，下一步仍需要先用 qPCR 驗證候選基因的 stage/tissue 表現，再選 MADS 或下游 target 做 VIGS/RNAi/functional assay。

分析邏輯

1. 先只使用我們自己的 CR0040A mapping 結果

原論文的公共 reads 被重新 mapping 到 CR0040_HaplotypeA。文章中的 HPP92/KAG 座標、WGCNA module 與 published candidates 先不納入 ranking，避免把文章結果直接帶進我們的排序；等 CR0040A 分析完成後，再獨立比較。

2. 用取樣設計定義 rostellum signal

GR_vs_G 是最接近 rostellum-specific 的比較，因為 GR 是含 rostellum 的蕊柱，G 是移除 rostellum 後的蕊柱。這個比較只在 S3/S4 有，因為原始取樣設計只有 S3/S4 同時具備 GR 與 G。

BGR_vs_B 可以從 S1 到 S4 做，因為每個 stage 都有完整花蕾 BGR 與移除蕊柱/rostellum 後的花被組織 B；它提供較粗略的 tissue-level support。

3. MADS 的定義採用 Pfam domain

本階段用 CR0040A annotation browser / Pfam 註解篩 MADS：PF00319（SRF-type / MADS-box DNA-binding domain）與 PF09047（MEF2 domain）作為 MADS core。PF01486（K-box）只當作 MIKC-type supporting annotation，單獨只有 K-box 不算 MADS core。

4. 最後用多個證據縮小候選

  • DESeq2-style differential expression：看 S3/S4 GR_vs_G 與 BGR_vs_B 是否支持。
  • MADS co-expression：看候選基因是否跟重要 MADS bait 同步變化。
  • OrchidBase7 predicted targets：看候選是否落在重要 MADS 的預測 target set。
  • Promoter CArG motif：作為可能被 MADS 直接調控的弱證據。
  • 功能註解：優先看器官生長、細胞壁 remodeling、糖分供應或發育調控相關基因。

下一步實驗

  • 先對 VANPL_A_00001t031260 與第一波 qPCR shortlist 做 S3/S4 tissue qPCR。
  • 確認 GR > G 或 rostellum-associated pattern 是否重現。
  • 從 qPCR 支持的 MADS / target 中挑 1-3 個做 VIGS/RNAi 或其他 knockdown assay。
  • 觀察 rostellum 大小、形態、授粉操作時間與結實率，避免只看分子表現。
